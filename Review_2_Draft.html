ABSTRACT:

Fake online reviews severely compromise consumer trust and e-commerce integrity. This project presents FakeScope, a novel 3-layer hybrid fake review detection system designed to identify deceptive opinion spam across domains (e.g., Amazon and Flipkart). Unlike existing solutions that rely solely on either text analytics or user metadata, FakeScope integrates both. It features a rule-based heuristic filter, a computationally efficient Hybrid Machine Learning model (combining TF-IDF vectors with stylometric features), and a Large Language Model (LLM) advisory layer. The ML model achieves an 87.2% cross-domain accuracy, significantly outperforming zero-shot deep learning baselines. The system outputs a composite, explainable "Trust Score" (0–100) and actionable purchasing advice via an intuitive Streamlit dashboard. Keywords: Fake Review Detection, Machine Learning, Natural Language Processing, Stylometry, Large Language Models, E-commerce Trust.

[1.] INTRODUCTION:

With the exponential growth of e-commerce, customer reviews have become the primary deciding factor for online purchases. However, this has led to the proliferation of "Deceptive Opinion Spam"—fake reviews written to falsely promote or maliciously demote products. Recognizing fake reviews is challenging for human readers and traditional sentiment analysis tools. FakeScope addresses this by analyzing not just what is said (sentiment), but how it is said (linguistics) and who is saying it (behavioral metadata), providing consumers with a clear, explainable metric of product authenticity.

[2.] PROBLEM STATEMENT:

Current fake review detection systems are often domain-specific (e.g., trained only on hotel reviews and failing on electronics) and lack explainability for the end-user. Furthermore, relying purely on text classification ignores vital context like purchase verification and helpfulness votes. There is a critical need for a cross-domain, interpretable system that aggregates multiple risk signals into a clear verdict for consumers.

[3.] OBJECTIVES:

  1. To develop a hybrid ML model combining linguistic text features and stylometric structures to detect deceptive reviews.
  2. To achieve robust cross-domain generalization (e.g., training on Yelp and successfully testing on Amazon/Flipkart data).
  3. To calculate an explainable "Trust Score" by dynamically weighting ML probabilities, behavioral metadata (Verified Purchase, Helpful Votes), and aspect-based sentiment.
  4. To integrate an LLM (Groq) to provide an automated, qualitative advisory verdict (Buy/Caution/Avoid).
  5. To deploy the system as an interactive, user-friendly batch-processing dashboard.

[4.] SCOPE OF THE PROJECT:

The project is scoped to scrape and analyze English-language reviews from major e-commerce platforms (specifically Amazon and Flipkart via HTML upload). It evaluates text stylometry, sentiment, and basic user metadata (star ratings, verification status). It does not currently implement real-time live scraping (due to CAPTCHA restrictions) nor complex reviewer network graph analysis.

[5.] PROPOSED SYSTEM:

The proposed system is a robust 3-Layer pipeline: * Layer 1 (Rule-Based): instantly flags explicitly spammy reviews (e.g., excessively short, high sentiment, high superlative count). * Layer 2 (Hybrid ML): Extracts TF-IDF text representations and concatenates them with 6 custom stylometric features (word count, exclamation ratio, CAPS ratio, sentence length, first-person pronoun ratio, sentiment intensity). A Logistic Regression classifier predicts the deception probability. * Layer 3 (Aggregation & LLM): Combines the ML probability with extracted HTML metadata (Verified badges, ratings) to generate a Trust Score (0-100). The Groq LLM API synthesizes this data into a final consumer verdict.

[6.1] FINDINGS IN LITERATURE SURVEY:

The literature reveals a historical divide between purely linguistic approaches and purely behavioral approaches. Linguistic models struggle with cross-domain generalization, while behavioral models are hard to implement without internal platform data. The primary finding is that a hybrid approach—combining text stylometry (to catch crowdsourced spam) with front-end behavioral signals (like 'Verified Buyer' badges)—is the most effective method for a third-party application like FakeScope.

[7]. METHODOLOGY:

  1. Data Collection & Cleaning: Aggregating standard datasets (Yelp NYC/ZIP) with scraped Amazon data. Text is lowercased, stripped of HTML/URLs, and standardized.
  2. Feature Engineering:
    • Vectorizing text using TF-IDF.
    • Extracting dense stylometric arrays (e.g., punctuation ratios, capitalization density).
    • Parsing behavioral metadata (helpful votes, verification) directly from raw HTML uploads via BeautifulSoup.
  3. Model Training: Training a Logistic Regression model via a sparse hstack of the engineered features. The model is validated using 5-fold Cross-Validation (achieving 87.2% accuracy).
  4. Trust Score Formulation: A weighted algorithm penalizes the product if reviews are classified as deceptive or if behavioral risks are present (e.g., 5-star reviews from unverified accounts).
  5. Deployment: Packaging the inference pipeline into an interactive Streamlit application.

[8.] SOFTWARE REQUIREMENTS:

Functional Requirements: * System must accept raw HTML text batches from Amazon/Flipkart. * System must extract individual reviews and their associated metadata (stars, date, helpful votes, verification). * System must classify each review as "Deceptive" or "Truthful". * System must calculate and display an overall Product Trust Score. * System must provide an Aspect-based sentiment breakdown (e.g., "Quality", "Price").
